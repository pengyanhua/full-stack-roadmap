# 领域驱动设计 (DDD)

DDD是一种软件设计方法论,通过领域模型驱动复杂业务系统的设计与实现。

## 目录

1. [战略设计](01_strategic_design.md) - 限界上下文、上下文映射
2. [战术设计](02_tactical_design.md) - 实体、值对象、聚合根
3. [事件风暴](03_event_storming.md) - 协作式建模工作坊
4. [DDD实战](04_ddd_in_practice.md) - 电商系统案例

## DDD核心概念

```
┌────────────────────────────────────────────────────┐
│              DDD 分层架构                          │
├────────────────────────────────────────────────────┤
│  用户界面层 (UI)        │  API/Web                │
│  ───────────────────────┼─────────────────────    │
│  应用层 (Application)   │  用例协调               │
│  ───────────────────────┼─────────────────────    │
│  领域层 (Domain)        │  业务逻辑 ⭐           │
│  ───────────────────────┼─────────────────────    │
│  基础设施层 (Infra)     │  数据库/外部服务        │
└────────────────────────────────────────────────────┘
```
